# Testing Glossary

This glossary provides standardized definitions for commonly used testing terminology.

---

## Test Case

A set of inputs, execution conditions, and expected results designed to verify a specific aspect of the system.

---

## Test Scenario

A high-level description of a functionality or workflow that should be tested.

---

## Test Condition

A specific feature, rule, or requirement that should be validated through testing.

---

## Defect

A flaw in a software component that causes the system to produce incorrect or unexpected results.

---

## Regression Testing

Testing performed to confirm that previously working functionality continues to work after changes.

---

## Acceptance Criteria

Conditions that must be satisfied for a feature to be accepted by stakeholders.

---

## Test Coverage

A measure describing how much of the system or requirements are exercised by tests.

---

## Test Environment

The hardware, software, and configuration used to execute tests.

---

## Test Data

Input data used during test execution.

---

## Exploratory Testing

An approach to testing where learning, test design, and test execution occur simultaneously.
